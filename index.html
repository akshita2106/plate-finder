⚙️ How the Algorithm Works

A step-by-step breakdown of the ANPR image processing pipeline.

1
Grayscale Conversion
Reduce color channels to simplify processing

The first step converts the input image from a 3-channel BGR (Blue-Green-Red) color space to a single-channel grayscale image. This reduces computational complexity while preserving the structural information needed for edge detection.

// Convert to grayscale using OpenCV.js
let gray = new cv.Mat();
cv.cvtColor(src, gray, cv.COLOR_RGBA2GRAY);

Each pixel's intensity is calculated as: Y = 0.299R + 0.587G + 0.114B, which weights green the highest since human vision is most sensitive to it.

2
Bilateral Filtering
Noise reduction while preserving edges

A bilateral filter smooths the image while keeping sharp edges intact. Unlike Gaussian blur which smooths everything uniformly, a bilateral filter considers both spatial distance and intensity difference — so edges between the plate and background remain crisp.

// Apply bilateral filter
let filtered = new cv.Mat();
cv.bilateralFilter(gray, filtered, 11, 17, 17);

Parameters: d=11 (diameter), sigmaColor=17 (color similarity range), sigmaSpace=17 (spatial extent).

3
Canny Edge Detection
Find boundaries between regions

The Canny edge detector finds rapid intensity transitions (edges) in the image. It uses a multi-stage algorithm: Gaussian smoothing → gradient computation → non-maximum suppression → hysteresis thresholding. The result is a binary image where white pixels represent detected edges.

// Canny edge detection with auto thresholds
let edges = new cv.Mat();
cv.Canny(filtered, edges, 30, 200);

Lower threshold (30) and upper threshold (200) control sensitivity. A high ratio (1:6) helps detect strong edges while suppressing noise.

4
Contour Detection & Filtering
Locate rectangular plate-shaped regions

Contours are extracted from the edge map and filtered based on geometric properties of number plates: rectangular shape (4 corners), aspect ratio between 2:1 and 6:1, and sufficient area. The largest qualifying contour is selected as the most likely plate.

// Find and filter contours
let contours = new cv.MatVector();
cv.findContours(edges, contours, hierarchy,
  cv.RETR_TREE, cv.CHAIN_APPROX_SIMPLE);

// Filter: 4-sided polygon with plate-like ratio
for (let i = 0; i < contours.size(); i++) {
  let approx = cv.approxPolyDP(contours.get(i), epsilon);
  if (approx.rows === 4 && aspectRatio >= 2 && aspectRatio <= 6)
    candidates.push(contours.get(i));
}
5
Plate Extraction & Enhancement
Crop, warp, and threshold for OCR

The detected plate region is cropped with padding, perspective-corrected if tilted, resized to 3x for better OCR, and then binarized using adaptive thresholding. Morphological closing fills small gaps in characters.

// Enhance for OCR
cv.resize(plate, resized, new cv.Size(w*3, h*3));
cv.adaptiveThreshold(resized, binary,
  255, cv.ADAPTIVE_THRESH_GAUSSIAN_C,
  cv.THRESH_BINARY, 11, 2);
6
OCR — Text Recognition
Read characters using Tesseract.js

The enhanced plate image is fed into Tesseract.js for Optical Character Recognition. We use PSM 7 (single text line mode) with a whitelist of A-Z and 0-9 for better accuracy. Post-processing cleans up noise characters and validates the format.

// Tesseract.js OCR
const result = await Tesseract.recognize(plateCanvas, 'eng', {
  tessedit_char_whitelist: 'ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789',
  tessedit_pageseg_mode: '7'  // Single line
});
const text = result.data.text.trim();

📚 DIP Concepts Documentation

Understanding the Digital Image Processing techniques used in PlateVision.

🌗
Grayscale Conversion
▼

Grayscale conversion transforms a color image into shades of gray. Each pixel's RGB values are combined into a single luminance value using the formula:

Y = 0.299 × R + 0.587 × G + 0.114 × B

This weighted average reflects human visual perception — our eyes are most sensitive to green light, which is why it has the highest weight. Converting to grayscale reduces data from 3 channels to 1, making subsequent operations 3× faster.

  • Input: 3-channel RGB image (24-bit per pixel)
  • Output: 1-channel grayscale image (8-bit per pixel)
  • Why: Most structural features (edges, shapes) don't depend on color
🧹
Bilateral Filtering
▼

A bilateral filter is an edge-preserving, noise-reducing smoothing filter. Unlike Gaussian blur which uses only spatial distance, bilateral filtering also considers the intensity difference between pixels:

BF[I]ₚ = (1 / Wₚ) Σ G_σs(‖p - q‖) · G_σr(|Iₚ - Iq|) · Iq

Where G_σs is the spatial Gaussian (nearby pixels have more influence) and G_σr is the range Gaussian (similar-intensity pixels have more influence). This means edges are preserved because pixels across an edge have very different intensities.

  • Key parameters: d (filter diameter), σ_color, σ_space
  • Effect: Smooths textures/noise while keeping plate boundaries sharp
  • Trade-off: Slower than Gaussian blur, but much better for ANPR
📐
Canny Edge Detection
▼

The Canny edge detector is a multi-stage algorithm that produces thin, precise edges:

  • Stage 1 — Gaussian Smoothing: Remove noise to avoid false edges
  • Stage 2 — Gradient Computation: Compute intensity gradients using Sobel operators to find direction and magnitude of change
  • Stage 3 — Non-Maximum Suppression: Thin edges to 1-pixel width by keeping only local maxima in the gradient direction
  • Stage 4 — Hysteresis Thresholding: Use two thresholds — strong edges (above upper) are kept, weak edges (between lower and upper) are kept only if connected to strong edges
G = √(Gx² + Gy²), θ = arctan(Gy / Gx)

The output is a binary image where white pixels represent edge locations. The dual-threshold approach makes Canny more robust than simple threshold-based edge detectors.

📦
Contour Detection & Polygon Approximation
▼

Contour detection traces the boundaries of connected white regions in a binary image. OpenCV's findContours returns a list of point sequences representing each closed boundary.

Polygon Approximation (Douglas-Peucker algorithm) simplifies contours by reducing the number of vertices while preserving shape:

ε = α × arcLength(contour)

Where ε is the maximum distance between the original contour and its approximation. For number plate detection, we set α = 0.018 and look for approximations with exactly 4 vertices (quadrilaterals).

  • RETR_TREE: Retrieves all contours in a hierarchical structure
  • CHAIN_APPROX_SIMPLE: Compresses horizontal/vertical/diagonal segments to endpoints only
  • Filtering: Aspect ratio 2–6, area > threshold, 4-sided polygon
🔄
Perspective Transform
▼

When a plate is viewed at an angle, it appears as a trapezoid rather than a rectangle. Perspective transform corrects this by mapping the 4 corner points of the detected plate to a rectangular output.

M = getPerspectiveTransform(srcPoints, dstPoints)

The 3×3 transformation matrix M maps each source point to its corresponding destination point. warpPerspective applies this transform to produce a front-facing view of the license plate.

  • Input: 4 corner points of the detected plate contour
  • Output: A flat, rectangular crop of the plate
  • Benefit: Dramatically improves OCR accuracy for angled plates
⬛
Adaptive Thresholding
▼

Unlike global thresholding which uses a single cutoff value for the entire image, adaptive thresholding computes a different threshold for each pixel based on its local neighborhood:

T(x,y) = μ(x,y) - C

Where μ(x,y) is the weighted mean of the neighborhood (Gaussian weighting) and C is a constant. Pixels brighter than their local threshold become white (characters), darker ones become black (background).

  • Block Size: 11 (neighborhood size for computing local mean)
  • C: 2 (constant subtracted from mean)
  • Advantage: Handles uneven lighting, shadows, and varying plate colors
🧱
Morphological Operations
▼

Morphological operations modify shapes in binary images using a structuring element (kernel). Two fundamental operations:

  • Dilation: Expands white regions — useful for filling small gaps in characters
  • Erosion: Shrinks white regions — useful for removing small noise dots
  • Opening: Erosion → Dilation — removes noise while preserving shape
  • Closing: Dilation → Erosion — fills small holes in characters

In PlateVision, we apply morphological closing with a small rectangular kernel to connect broken character segments, making OCR more reliable.

🔤
Optical Character Recognition (OCR)
▼

OCR converts images of text into machine-readable strings. Tesseract uses a multi-step approach:

  • Layout Analysis: Identifies text regions and lines (PSM 7 = single text line)
  • Character Segmentation: Isolates individual characters from the text line
  • Feature Extraction: Extracts shape features from each character
  • Recognition: Uses LSTM neural networks trained on fonts to classify characters
  • Post-Processing: Dictionary lookup and language model for correction

For license plates, we restrict the character set to A-Z and 0-9 (whitelist), which eliminates common misreadings like O→0 confusion and dramatically improves accuracy.
